# Social Sharing Workspace

This directory tracks how Knowledge Pools can be shared publicly when social content is requested.

The goal is to keep a lightweight sharing framework ready without making social publishing part of every project step. Each requested update should connect one concrete project step to one useful idea for readers.

## Current Mode

Social content generation is on demand.

Do not create new stage-specific posts automatically. Use this directory as a reference and generate content only when the user asks for it.

## Files

- [strategy.md](strategy.md): audience, tone, channels, and content principles.
- [stage-plan.md](stage-plan.md): project stages and what to publish at each step.
- [stage-0-problem-approach.md](stage-0-problem-approach.md): first public narrative about the problem and approach.
- [stage-1-ingest-baseline.md](stage-1-ingest-baseline.md): second public narrative about the ingest architecture baseline.
- [stage-2-understand-baseline.md](stage-2-understand-baseline.md): third public narrative about the understand architecture baseline.
- [on-demand-workflow.md](on-demand-workflow.md): rules for creating social content only when requested.
- [post-template.md](post-template.md): reusable format for drafting posts.
- [drafts.md](drafts.md): existing draft posts and reference material.

## Publishing Rule

Before publishing any post, check:

1. Is the claim grounded in actual repository progress?
2. Does it explain one idea clearly?
3. Does it invite useful conversation?
4. Does it avoid pretending the system is more complete than it is?
